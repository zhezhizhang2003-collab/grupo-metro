## Codebase Patterns

- **Diseño Industrial Bold:** negro (#0a0a0a, bg-zinc-950, bg-black) + ámbar (#f59e0b / amber-400)
- **Fuentes:** Bebas Neue (display headings) + Inter (body 400-700)
- **Tailwind v4:** usa `@import "tailwindcss"` y `@theme` en CSS (no tailwind.config.js)
- **Astro 6:** soporta Tailwind v4 nativamente con `@tailwindcss/vite`
- **Marquee animation:** CSS keyframes con `translateX(-50%)`, array duplicado para bucle continuo
- **Logo naming:** `Coca-Cola.png`, `Estrella-Damm.png` — espacios → guiones, caracteres especiales eliminados
- **Layout.astro:** Componente layout que acepta `title` y `description` via Astro.props
- **Sin componentes separados:** Los componentes están inline — refactorizar a `src/components/` cuando crezca
- **PDFs en /public/:** acceso directo sin procesamiento, no pasan por Astro build pipeline

**Learnings for future iterations:**
- SVG vectorial siempre preferible sobre PNG para logos (worldvectorlogo.com buena fuente)
- sips en macOS convierte JPEG→PNG correctamente, pero no mejora la calidad del original
- Usar OCR (tesseract) para identificar imágenes sin nombre
- Los PNG con paleta de 8-bit (colormap) pueden perder calidad — mejor RGBA
- Tailwind v4 usa `@theme` en CSS, no `tailwind.config.js`
- El menú móvil animado necesita max-height calculado con scrollHeight para la transición
- Siempre verificar MD5 de las imágenes para detectar duplicados

**Codebase Patterns actualizadas:**
- SVG logos en `public/logos/` con nombres: espacios→guiones, sin tildes
- Layout.astro ahora incluye OpenGraph + Twitter Card + JSON-LD + canonical automáticos
- El array `marcas` en index.astro es la fuente de verdad para la marquee
- `logoUrl()` usa dos arrays: `logosSvg` y `logosPng` — actualizar ambos al añadir/quitar logos
- Mobile menu script inline en Layout.astro con animación CSS + JS
